Seksi Commander dokumentace

Introduction

Seksi Commander is a filemanager for X11 and Linux. In this moment is only x86 version available. Seksi commander is inspired by mix of Total Commander, Midnight Commander and my imagine.

Development info

Seksi Commander is written in Object Pascal (Kylix and now in FPC+Lazarus).

License

Program Seksi Commander is released under terms of GNU GPL v 2.

For internal editor Seksi Commander use SynEdit components, which is released under Mozilla Public License or/and GNU GPL 2 (included in Lazarus).

For integrated viewer I wrote whole new component ViewerControl (mmaped bin, hex, text mode) and released under same licence as Seksi Commander (GNU GPL 2).

Why new commander?

Main target is develop file manager for X11 and Linux with this features (marked with * is currently not implemented):
  • searching with ALT+letter
  • two panels
  • sort by filename, file extension, file size ...
  • file operation with threads (and in near future at background) - file copy, file move, delete
  • work transparently with archives (in this moment only browsing, copying, moving and deleting I planed about 5.7.2003)
  • lynx like directory movement
  • integrated editor with syntax hightlighting
  • integrated file viewer (text, hex, bin, graphics)
  • independ to KDE and Gnome
Other features:
  • icons
  • Multi rename tool
  • selecting files by INS
  • HotDir
  • * FTP
  • can use external editor, viewer or differ
  • Directory history (last 50)
  • Shift+F6 like Total Commander
  • configure operation on file (by definition in sc.ext)
  • colorize items in panels by file ext
  • multi language support (in this moment:)
    • English
    • Czech
    • Slovak
  • and many other

If you need anything else, try to write or write wish to mailing list :-).

Viewer

Integrated File Viewer have this modes: (text, bin, hex, wrapped text, processing file).

Also can directly show this images :JPG, PNG, BMP, ICO, XPM, TGA, VMF.

If you select more than one file, can browse selected files with keys N(ext), P(rior) (like Lister in Total Commander). Switch file showing mode with menu or keys 1, 2, 3, 4, (5 is currently disabled).

Program access files throught mmap (text modes) or for graphics load file to memory.

Text mode

In this mode viewer show file as text. ASCII codes <32 are showed as spaces, only ASCII code 9 is replaced by some spaces (by current cursor position). Code 10 and code 13 is interpreted as new line. If line is too long, program wrap text and continued on next line.

Text wrapped text

Like text mode, but with wrapping on 80 chars.

Mode bin

In this mode all codes <32 are replaced with space and wrap text at 80 chars.

Mode hex

Show file in hex. Also show ASCII codes in right column.

Mode process

(currenly disabled) Program search in file sc.ext for View command and display result (result is readed from stdout of this command). If View command not found, mode is not changed.

Editor

Integrated editor have keyboard mapping aka Borlands IDE. I use SynEdit components. This component package is written for Delphi and Kylxi port is only few months old. In windows with this components are writen many programs (like PSPad, HydraPHP ...).

If you can other syntax highlighting that includes in editor, must use synedit generator (part of synedit package). Generator is based on lex.

In new version is implemented search and replace (from synedit demos).

Multi rename tool

This dialog is inspired (but written from scratch) by Total Commander and total redesigned.

Compare files dialog

Work in two modes: bin, text. Written in scratch.

How can I send bugreport

At first moment see status if your bug is known. For some of bugs (most not my) I implement temporally workaround what may not worked at every system (see dead key problem with synedit).

In release version is a bug too if no shortcut (on button, menu ...) or any feature is not available without mouse. Program must be controled without mouse!

In development version send only critical error (crash, error messages, hang up, strange message and so).

Send bugs (and wish) to mailing list (in english or in czech) see authors and contacts.

How can install Seksi Commander?

Exists two ways: compile source or install binary.

For binary instalation download file develxxxxx.tgz. Where xxxxx is something other than xxxxx :-). develxxxx is development package and at this moment not exists any release package.

Simple unpack files from archive where you want and execute sc. In this development version package is only unpacked to any directory. At release version will be changed.

Files of Seksi Commander

Main program is sc. There is no starting script(!).

In lng subdirectory are language files. Seksi commander can use any file with lng extension (no other registration is needed).

In main directory (from version 0.4) exists file sc.ext (based on mc.ext). In this file are descripted actions for any extension (for popup menu, execute or viewing ...)

Definitions of file colors in panel are in color.ext, format of file is descripted at begin (very simple).

In file pixmaps.txt are defined pixmaps for files.
